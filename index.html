Interactive Scenario

Real hujum qanday sodir bo'ladi?

Haqiqiy kiber hujum stsenariysi — bosqichma-bosqich

STEP 01
🔍 Razvedka (Reconnaissance)
Hacker nishon kompaniya haqida ma'lumot yig'adi: domenlar, email manzillar, xodimlar, texnologiyalar. OSINT va passiv usullar qo'llaniladi.
STEP 02
📧 Phishing Email
Ishonchli manbadan kelgandek ko'rinadigan email jo'natiladi. Ichida zararli PDF fayl yoki havola mavjud bo'ladi.
STEP 03
📄 Foydalanuvchi faylni ochadi
Qurbon PDF yoki hujjatni ochadi. Odatiy ko'rinishdagi fayl orqa fonda zararli kodni ishga tushiradi.
STEP 04
💀 Malware bajarilishi
Zararli dastur tizimga o'rnatiladi, backdoor yaratiladi, hujumchi tizimni nazorat qila boshlaydi. Antivirus ko'pincha bu bosqichda aniqlay olmaydi.
STEP 05
📤 Ma'lumotlar o'g'irlanishi
Maxfiy ma'lumotlar, parollar, moliyaviy hujjatlar hujumchi serveriga yuboriladi. Ko'pincha bu haftalab yashirin qoladi.
Curriculum

Nima o'rganasiz?

Industry-standard ko'nikmalar va real dunyoda ishlatiladigan texnikalar

🎯
Reconnaissance
OSINT, subdomain enumeration, passive/active recon texniikalari
💉
SQL Injection
In-band, blind, time-based SQLi. SQLMap va manual exploitation
⚡
XSS
Reflected, stored, DOM-based XSS. Bypass va real impact
🔄
SSRF
Server-Side Request Forgery. Cloud metadata, internal services
📄
XXE
XML External Entity injection. File read, SSRF via XXE
🔌
API Testing
REST/GraphQL API security testing. Auth bypass, IDOR
🏆
Bug Hunting
Bug bounty platforms: HackerOne, Bugcrowd. Report yozish
🔑
Auth Vulnerabilities
JWT attacks, OAuth misconfigurations, session management
